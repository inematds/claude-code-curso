⚡ Aliases de shell para produtividade

🧠 Imagine assim: aliases transformam comandos longos em atalhos de 2-3 letras. Cada segundo economizado se multiplica por centenas de usos ao longo de semanas.

O primeiro passo para acelerar seu workflow e criar aliases no seu .bashrc ou .zshrc. O Claude Code e um comando que voce vai digitar dezenas de vezes por dia. Encurtar claude para cc ja economiza 4 caracteres por uso.

Alem do alias basico, crie aliases para operações compostas que voce repete frequentemente: resume a ultima sessão, rode em modo one-shot com modelo especifico, ou inicie numa pasta especifica.

aliases recomendados (.bashrc / .zshrc)
# Aliases para Claude Code
alias cc='claude'
alias ccr='claude --resume'
alias ccp='claude -p'
alias cco='claude --model opus'
alias ccs='claude --model sonnet'

# Alias com pipe útil
alias ccgit='git diff | claude -p "resuma estas mudanças"'
alias ccreview='claude -p "revise este arquivo" <'

Em 1 frase: aliases como cc, ccr, ccp encurtam comandos e aceleram o dia a dia.

2

📜 Scripts wrapper personalizados

🧠 Imagine assim: scripts wrapper combinam claude -p com outras ferramentas numa operação unica. E automação caseira: simples, util, e so sua.

Um wrapper e um script de shell que combina o Claude Code com outras ferramentas. Diferente de um alias (que e so um atalho), um wrapper pode ter logica: verificar condições, processar a saida, salvar em arquivo.

Exemplos praticos: um script que roda git diff, passa pro Claude Code revisar, e salva o resultado num arquivo Markdown. Ou um script que pega um arquivo de log, manda pro Claude Code diagnosticar, e abre o resultado no editor.

exemplo: review-pr.sh
#!/bin/bash
# review-pr.sh - Review automatizado de PR
DIFF=$(git diff main...HEAD)
if [ -z "$DIFF" ]; then
  echo "Nenhuma mudança encontrada."
  exit 0
fi
echo "$DIFF" | claude -p "Revise este diff.
Aponte: bugs, melhorias, e se segue os padroes do projeto.
Formato: markdown com seções." > review.md
echo "Review salvo em review.md"

Em 1 frase: scripts wrapper = claude -p + logica de shell para automatizar tarefas repetitivas.

3

💻 Integração com VS Code

🧠 Imagine assim: o VS Code e o editor mais popular. Usar o Claude Code dentro dele (terminal integrado + extensão) cria um workflow onde voce nunca precisa sair do editor.

A forma mais simples e rodar o Claude Code no terminal integrado do VS Code (Ctrl+`). Voce ve o codigo em cima e conversa com o Claude embaixo. Melhor ainda: use dois painéis lado a lado: editor + terminal com Claude Code.

A Anthropic tambem oferece uma extensão oficial do Claude Code para VS Code que integra mais profundamente: o agente aparece como um painel lateral, pode referenciar arquivos abertos, e as edições aparecem em tempo real no editor.

VS Code settings sugeridas
{
  "terminal.integrated.defaultProfile.linux": "bash",
  "terminal.integrated.fontSize": 13,
  "editor.formatOnSave": true,
  "workbench.panel.defaultLocation": "right"
}

Em 1 frase: terminal integrado do VS Code + extensão oficial = Claude Code sem sair do editor.

4

🪟 Multiplexer: tmux/screen + Claude

🧠 Imagine assim: tmux permite rodar o Claude Code em sessões que sobrevivem a desconexão. Fechou o terminal? O Claude continua rodando. Reconecte quando quiser.

O tmux (ou screen) e o parceiro perfeito do Claude Code para quem trabalha em servidores remotos ou quer sessões que nao morrem. Voce abre um pane com o Claude Code, outro com seu editor, e pode desconectar e reconectar a qualquer momento.

Um setup classico: tmux new-session -s dev, divide em dois panes (Ctrl+B %), roda claude num e edita no outro. Se precisar sair, Ctrl+B d desconecta sem matar. tmux attach -t dev reconecta.

tmux + Claude Code
# criar sessão
tmux new-session -s claude-dev

# dividir em dois panes
Ctrl+B %     # split vertical
Ctrl+B "     # split horizontal

# rodar claude no pane esquerdo
claude

# desconectar (claude continua rodando)
Ctrl+B d

# reconectar depois
tmux attach -t claude-dev

Em 1 frase: tmux cria sessões persistentes que sobrevivem a desconexão; ideal para trabalho remoto.

5

📦 Dotfiles e portabilidade

🧠 Imagine assim: dotfiles versionados no git garantem que voce reproduz seu setup em qualquer maquina nova em minutos. Aliases, configs, CLAUDE.md global: tudo versionado.

Crie um repositorio de dotfiles com: .bashrc/.zshrc (aliases), ~/.claude/CLAUDE.md (preferências globais), ~/.claude/settings.json (config global), e um install.sh que cria os symlinks.

Com dotfiles versionados, formatar a maquina ou mudar de computador nao e mais trauma. Clone o repo, rode o script de instalação, e em 5 minutos seu Claude Code esta configurado exatamente como antes.

💡 Estrutura sugerida de dotfiles

dotfiles/ → bash/aliases.sh, claude/CLAUDE.md, claude/settings.json, tmux/.tmux.conf, install.sh. O install.sh cria symlinks de cada arquivo para a localização correta no home.

Em 1 frase: dotfiles no git = setup reproduzivel em qualquer maquina nova.

6

🏆 O daily driver setup definitivo

🧠 Imagine assim: aqui juntamos tudo: aliases + VS Code + tmux + dotfiles + CLAUDE.md global. O resultado e um setup que voce liga de manhã e esta produtivo em 10 segundos.

O setup completo une todas as peças: aliases no shell para velocidade, VS Code com terminal integrado para contexto visual, tmux para sessões persistentes, dotfiles para portabilidade, e CLAUDE.md global para preferências que valem em todo projeto.

O ritmo diario ideal: abrir o tmux, rodar ccr (alias para claude --resume) para retomar onde parou, trabalhar por 1-2 horas, commitar checkpoints, e quando o contexto encher, iniciar sessão nova. Cada dia voce melhora o setup: um alias novo aqui, uma regra no CLAUDE.md ali. Em uma semana, o workflow ja esta visivelmente mais rapido que no dia 1.

daily-driver-checklist.txt
Setup diário com Claude Code:
[x] Aliases no .bashrc/zshrc (cc, ccr, ccp)
[x] VS Code com terminal integrado
[x] tmux para sessões persistentes
[x] CLAUDE.md global com preferências pessoais
[x] CLAUDE.md por projeto com stack e regras
[x] .claudeignore protegendo secrets
[x] settings.json com allowedTools afinados
[x] Dotfiles no git para portabilidade
[x] Ritmo: resume → trabalha → commit → repete

Em 1 frase: aliases + VS Code + tmux + dotfiles + CLAUDE.md = produtividade maxima em 10 segundos.
