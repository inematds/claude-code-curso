⚡ Atalhos de teclado essenciais

🧠 Imagine assim: atalhos de teclado no Claude Code sao como os atalhos no Vim. No começo voce ignora, depois de aprender nao vive sem. A diferença e que aqui sao apenas 5-6 teclas.

Dentro da sessão interativa do Claude Code, algumas teclas tem comportamento especial que acelera drasticamente seu workflow. Os mais importantes sao poucos e faceis de memorizar:

✓ Atalhos essenciais

  • • Escape — cancela a ação corrente / volta ao prompt
  • • Tab — aceita sugestão / autocomplete
  • • Ctrl+C — interrompe a geração em andamento
  • • Shift+Tab — rejeita sugestão
  • • Enter duplo — envia mensagem multi-linha

→ Dicas de navegação

  • • Seta para cima — editar ultima mensagem
  • • Ctrl+L — limpar tela (sem perder contexto)
  • • / no inicio — abre slash commands
  • • @ — referenciar arquivo
  • • Colar imagem — analise visual automatica

Em 1 frase: Escape, Tab, Ctrl+C, Shift+Tab e Enter duplo cobrem 95% da interação.

2

🔧 Slash commands

🧠 Imagine assim: slash commands sao o painel de controle escondido do Claude Code. Digitou /, apareceu um menu. E como o command palette do VS Code, mas dentro da conversa com a IA.

Dentro de uma sessão interativa, digitar / no inicio da mensagem ativa os slash commands. Eles controlam a sessão sem gastar tokens de conversa. Os mais usados:

slash commands essenciais
/help       — lista todos os comandos disponíveis
/clear      — limpa o contexto da conversa atual
/compact    — comprime o contexto para liberar espaço
/model      — ver ou trocar o modelo em uso
/memory     — salvar uma nota persistente
/resume     — retomar sessão anterior
/cost       — ver quanto gastou na sessão atual

/help

O ponto de partida. Mostra tudo que esta disponivel. Rode quando estiver perdido.

/clear

Apaga todo o contexto. Util quando voce quer recomeçar sem fechar o terminal.

/model

Mostra qual modelo esta ativo. Pode trocar entre Opus e Sonnet sem sair da sessão.

/memory

Salva uma preferência que persiste entre sessões. Exemplo: "sempre use TypeScript".

Em 1 frase: digite / para acessar o painel de controle do Claude Code dentro da sessão.

3

🧠 Modelo e janela de contexto

🧠 Imagine assim: o modelo e o cerebro; a janela de contexto e a memória de trabalho. Imagine que o Claude tem uma mesa de trabalho com espaço limitado. Tudo que voce diz e tudo que ele le vai pra mesa. Quando enche, algo cai.

O Claude Code usa modelos da familia Claude da Anthropic. O padrao costuma ser o Claude Sonnet (rapido e capaz), mas voce pode mudar para Claude Opus (mais profundo e preciso) a qualquer momento. A janela de contexto e o limite do que o modelo ve ao mesmo tempo: tudo que voce disse, tudo que ele leu, todos os arquivos inspecionados.

📊 Números que importam

O context window do Claude e de aproximadamente 200k tokens (cerca de 150k palavras). Parece muito, mas um codebase medio consome isso rapido quando o agente le varios arquivos. Cada arquivo lido, cada resposta gerada, cada CLAUDE.md carregado come tokens dessa janela.

Em 1 frase: Sonnet = rapido, Opus = profundo; 200k tokens e a mesa de trabalho e tudo compete por espaço.

4

💾 Sessões e /resume

🧠 Imagine assim: fechar o terminal nao apaga a conversa. E como fechar um livro com marcador. Quando voce volta, o /resume abre na pagina certa.

O Claude Code persiste sessões automaticamente. Cada conversa tem um session ID e fica salva localmente. Para retomar, voce pode usar claude --resume na linha de comando ou /resume dentro de uma sessão nova. O Claude vai mostrar um picker com as sessões recentes para voce escolher.

retomando sessões
# retomar a última sessão
claude --resume

# dentro de uma sessão, retomar outra
/resume

# continuar conversa específica por ID
claude --resume SESSION_ID

Em 1 frase: sessões persistem no disco; --resume ou /resume retoma de onde voce parou.

5

🖼️ Leitura de imagens e screenshots

🧠 Imagine assim: voce pode colar um screenshot de um bug no terminal e o Claude Code vai analisar visualmente, identificar o problema e sugerir o fix. E como ter um par de olhos extra que lê codigo na tela.

O Claude Code e multimodal. Ele pode ler imagens de tres formas: colar do clipboard (Ctrl+V com um screenshot copiado), referenciar um arquivo de imagem no filesystem, ou arrastar e soltar no terminal (em terminais que suportam). Casos de uso comuns: debug visual de UI, reproduzir um design de um mockup, ler diagramas de arquitetura.

⚠️ Limitações

Imagens consomem tokens (e bastante). Um screenshot pode usar 1000+ tokens. Use com moderação e prefira descrever em texto quando possivel. O Claude Code tambem nao consegue editar imagens, apenas analisar.

Em 1 frase: cole screenshots, passe paths de imagem ou arraste arquivos e o Claude analisa visualmente.

📤 Output formatado e pipes

🧠 Imagine assim: o mode -p transforma o Claude Code em um cidadão Unix de primeira classe. Tudo o que ele responde vai pro stdout, pronto para pipar com jq, grep, ou redirecionar para um arquivo.

A integracao com o ecossistema Unix e um dos superpoderes do Claude Code. Com claude -p, a saida vai diretamente pro stdout, permitindo encadear com qualquer ferramenta Unix:

exemplos de pipes
# gerar JSON e processar com jq
claude -p "liste 5 tarefas em JSON" | jq '.[]'

# analisar um arquivo e salvar resultado
claude -p "revise este código" < app.py > review.md

# pipe de entrada: mandar conteúdo para análise
git diff | claude -p "resuma estas mudanças"

# output em JSON estruturado
claude -p "..." --output-format json

Em 1 frase: claude -p + pipes Unix = IA como ferramenta de linha de comando composivel.
