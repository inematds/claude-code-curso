📋 Requisitos e instalação

🧠 Imagine assim: instalar o Claude Code é como instalar o Git pela primeira vez. Um comando, uma autenticação, e a partir dai voce tem um novo superpoder no terminal que nunca mais vai largar.

O Claude Code é uma ferramenta de linha de comando (CLI) da Anthropic que coloca o Claude diretamente no seu terminal. Diferente de chat web, aqui o Claude tem acesso ao seu codebase, pode ler e editar arquivos, rodar comandos e agir como um agente de desenvolvimento completo.

Para instalar, voce precisa de Node.js 18+ (o runtime JavaScript). Se ja tem Node, a instalação é um unico comando. Existem tres caminhos, dependendo do seu sistema operacional:

instalação via npm (qualquer SO) 
npm install -g @anthropic-ai/claude-code

npm (universal)

Funciona em Mac, Linux e Windows. Precisa de Node.js 18+ instalado. O caminho mais comum.

Homebrew (macOS)

brew install claude-code se voce usa Homebrew no Mac. Gerencia atualizações automaticamente.

winget (Windows)

winget install Anthropic.ClaudeCode no Windows. Integra com o gerenciador de pacotes nativo.

💡 Dica: autenticação

Na primeira execução, o Claude Code vai pedir autenticação. Voce pode usar login via browser (recomendado) ou setar a variavel ANTHROPIC_API_KEY no seu ambiente. O login via browser vincula sua conta Anthropic e habilita o uso com os planos Pro/Max/Team.

Em 1 frase: Node.js 18+ e um npm install -g e voce tem o Claude Code funcionando.

2

▶️ Primeiro claude no terminal

🧠 Imagine assim: abrir o Claude Code pela primeira vez é como ligar a ignição de um carro novo. O motor ronca, o painel acende, e voce percebe que tem um copiloto inteligente no banco do passageiro.

Abra o terminal, navegue ate a pasta de qualquer projeto e digite claude. Pronto. A interface interativa vai aparecer, mostrando o modelo que esta rodando e esperando seu prompt. Voce esta conversando diretamente com o Claude, mas agora ele tem acesso aos arquivos do diretorio corrente.

primeiro comando 
cd ~/meu-projeto
claude

Ao entrar, experimente pedir algo simples: "explique o que faz este projeto" ou "liste os arquivos principais". O Claude vai ler os arquivos do diretorio e responder com contexto real. Isso ja e diferente de qualquer chatbot web, porque ele esta dentro do seu projeto.

one-shot mode (sem sessão) 
claude "explique o que faz o arquivo package.json"

Em 1 frase: claude no terminal abre uma sessão interativa com acesso total ao seu projeto.

3

🔀 Modo interativo vs one-shot

🧠 Imagine assim: o modo interativo e como sentar numa mesa com o Claude para trabalhar junto por horas. O one-shot e como mandar um recado rapido pela janela e receber a resposta de volta.

O Claude Code tem dois modos fundamentais de uso. No modo interativo (so digitar claude), voce abre uma conversa continua. Cada mensagem mantem o contexto das anteriores, o Claude pode pedir confirmação antes de editar arquivos, e voce controla o fluxo passo a passo. E o modo padrao para desenvolvimento.

No modo one-shot (usando a flag -p), voce manda um unico prompt, recebe a resposta e o processo encerra. Ideal para automação, scripts e perguntas rapidas. A flag -p (de "print") diz: execute isso e me devolva o resultado no stdout.

✓ Modo interativo

  • • Conversa continua com contexto
  • • Pede confirmação para edições
  • • Ideal para desenvolvimento
  • • Comando: claude

→ Modo one-shot

  • • Uma pergunta, uma resposta
  • • Saida vai pro stdout
  • • Ideal para scripts e pipes
  • • Comando: claude -p "..."

Em 1 frase: claude para conversar, claude -p para perguntar e sair.

4

📖 Navegando o --help e flags essenciais

🧠 Imagine assim: as flags do Claude Code sao como os botoes do painel de um avião. Voce nao precisa saber todos no primeiro dia, mas os essenciais mantem voce no ar.

O ponto de partida e sempre claude --help. Ele lista todas as flags disponiveis com uma explicação curta. As que voce vai usar no dia a dia sao poucas:

flags essenciais 
claude --help          # ver todas as opções
claude --version       # versão instalada
claude -p "..."        # modo one-shot (print)
claude --resume        # retomar sessão anterior
claude --model opus    # escolher o modelo
claude --verbose       # modo verboso (debug)

--help

Sempre o primeiro comando quando voce esquece alguma coisa. Referencia completa na ponta dos dedos.

-p (print / one-shot)

A flag mais usada. Transforma o Claude Code em ferramenta de pipe Unix.

--resume

Retoma a sessão anterior. Essencial quando voce fecha o terminal sem querer.

--model

Escolhe qual modelo usar. Opus para tarefas complexas, Sonnet para tarefas rapidas.

Em 1 frase: --help e seu manual de bolso; -p, --resume e --model sao as flags do dia a dia.

5

🔄 Atualizando e versionando

🧠 Imagine assim: o Claude Code evolui rapido. Cada versão nova traz features, melhorias e fixes. Manter atualizado e como manter a caixa de ferramentas afiada.

O Claude Code recebe atualizações frequentes. Verificar sua versão e atualizar sao operações simples:

verificar e atualizar 
claude --version                              # ver versão atual
npm update -g @anthropic-ai/claude-code       # atualizar via npm

O Claude Code tambem avisa quando uma atualização esta disponivel ao iniciar uma sessão. A recomendação e manter sempre a versão mais recente, ja que features novas (como MCP servers, melhores permissões e novos slash commands) aparecem frequentemente. Consulte o changelog oficial para saber o que mudou entre versões.

💡 Sobre breaking changes

Grandes mudanças sao raras, mas acontecem. Se voce tem scripts que dependem do Claude Code, teste apos atualizar. O comando claude --version no inicio de scripts ajuda a garantir compatibilidade.

Em 1 frase: npm update -g para atualizar; --version para conferir.

6

🔧 Diagnóstico de problemas comuns

🧠 Imagine assim: como todo software, o Claude Code pode dar problema na instalação. A boa noticia e que os erros mais comuns tem soluções de 1 linha.

Se algo deu errado na instalação ou na primeira execução, passe por este checklist. Ele cobre 90% dos problemas que iniciantes encontram:

✗ Erros comuns

  • • EACCES permission denied no npm install -g
  • • Node version mismatch (precisa 18+)
  • • Authentication failed na primeira execução
  • • command not found: claude apos install

✓ Soluções

  • • Use sudo npm install -g ou configure npm prefix
  • • Atualize Node: nvm install 20
  • • Rode claude e siga o login no browser
  • • Verifique se npm/bin esta no PATH
checklist-diagnostico.txt 
Claude Code não funciona? Checklist rápido:
[ ] Node.js 18+ instalado?  → node --version
[ ] npm global acessível?   → npm list -g --depth=0
[ ] claude no PATH?         → which claude
[ ] Autenticação ok?        → claude (e siga o login)
[ ] Versão atualizada?      → claude --version
Se 3+ falharam, reinstale: npm uninstall -g @anthropic-ai/claude-code && npm install -g @anthropic-ai/claude-code

Em 1 frase: Node, npm, PATH e auth: resolva esses quatro e o Claude Code funciona.
