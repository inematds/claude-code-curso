📦 Instalação e Primeiro Comando

Do npm install ao primeiro prompt funcional. Tudo que você precisa pra começar a usar o Claude Code hoje.

O que é:

As três formas de instalar o Claude Code: npm, Homebrew e winget.

Por que aprender:

Sem instalação correta, nada mais funciona.

Conceitos-chave:

Node.js 18+; npm global install; autenticação com API key.

O que é:

Rodar o comando claude pela primeira vez e entender o que aparece.

Por que aprender:

O primeiro passo real. Sem ele, o resto é teoria.

Conceitos-chave:

Interface interativa; prompt do Claude; resposta inline.

O que é:

claude (interativo) vs claude -p "pergunta" (one-shot, sem sessão).

Por que aprender:

Saber quando usar cada modo economiza tempo e contexto.

Conceitos-chave:

Modo interativo = conversa; flag -p = one-shot; pipes com -p.

O que é:

As flags que voce vai usar todo dia: --help, --version, -p, --model, --verbose.

Por que aprender:

Saber as flags evita ficar perdido no Google.

Conceitos-chave:

--help como referência rápida; flags mais usadas.

O que é:

Como atualizar o Claude Code e verificar a versão instalada.

Por que aprender:

Versões novas trazem features e fixes importantes.

Conceitos-chave:

npm update -g; changelog; breaking changes.

O que é:

Os erros mais comuns na instalação e como resolver cada um.

Por que aprender:

Saber resolver erros impede que você trave logo no começo.

Conceitos-chave:

EACCES permission; Node version mismatch; auth errors.

⌨️ Navegando o CLI como um Pro

Atalhos de teclado, slash commands, sessões e pipes que multiplicam sua velocidade.

O que é:

Os atalhos de teclado do Claude Code: Escape, Tab, Ctrl+C e mais.

Por que aprender:

Atalhos economizam cliques e mantêm o foco no terminal.

Conceitos-chave:

Escape para cancelar; Tab para aceitar; Ctrl+C para interromper.

O que é:

Comandos com barra dentro da sessão interativa do Claude Code.

Por que aprender:

São o painel de controle dentro da conversa.

Conceitos-chave:

/help; /clear; /model; /memory; /resume; /compact.

O que é:

Qual modelo roda por trás do Claude Code e como o contexto funciona.

Por que aprender:

Entender o modelo ajuda a calibrar expectativas e custos.

Conceitos-chave:

Opus/Sonnet; context window; token budget.

O que é:

Sessões persistem entre execuções. /resume retoma a conversa anterior.

Por que aprender:

Não perder o contexto de trabalho ao fechar o terminal.

Conceitos-chave:

Session ID; /resume; --resume flag; session picker.

O que é:

Claude Code lê imagens do filesystem e analisa screenshots.

Por que aprender:

Debug visual, reproduzir designs, ler diagramas.

Conceitos-chave:

Drag-and-drop; colar screenshot; referência por path.

O que é:

Usar claude -p com pipes Unix: | jq, | grep, > arquivo.

Por que aprender:

Conceitos-chave:

🧠 CLAUDE.md — O Cerebro do Projeto

O arquivo que transforma o Claude Code de genérico em especialista no seu projeto.

O que é:

Um arquivo Markdown que o Claude Code lê automaticamente a cada sessão.

Por que aprender:

É o único jeito de dar contexto persistente ao agente sobre seu projeto.

Conceitos-chave:

System prompt; instruções persistentes; contexto de projeto.

O que é:

~/.claude/CLAUDE.md (global) > ./CLAUDE.md (projeto) > subdir/CLAUDE.md (pasta).

Por que aprender:

Cada nível resolve um problema diferente de escopo.

Conceitos-chave:

Cascata de instruções; herança; override por nível.

O que é:

Como escrever CLAUDE.md que o modelo realmente segue.

Por que aprender:

Um CLAUDE.md mal escrito é ignorado; um bom muda tudo.

Conceitos-chave:

Headings claros; listas; exemplos; tom imperativo.

O que é:

Quais tipos de instrução realmente mudam o comportamento do agente.

Por que aprender:

Menos instruções certas > muitas instruções vagas.

Conceitos-chave:

Stack; convenções; proibições; padrões do projeto.

O que é:

Exemplos comentados de CLAUDE.md de projetos reais.

Por que aprender:

Ver exemplos reais encurta a curva de aprendizado.

Conceitos-chave:

Patterns comuns; anti-patterns; tamanho ideal.

O que é:

CLAUDE.md deve evoluir junto com o projeto, e ir no git.

Por que aprender:

Um CLAUDE.md desatualizado é pior que nenhum.

Conceitos-chave:

Commitar no repo; revisar mensalmente; /memory como complemento.

O que é:

O arquivo JSON que controla comportamento global e por projeto.

Por que aprender:

Sem settings.json voce fica com os defaults genéricos.

Conceitos-chave:

~/.claude/settings.json; .claude/settings.json; merge de configs.

O que é:

Arquivo que diz ao Claude Code quais arquivos/pastas ignorar.

Por que aprender:

Protege secrets, reduz contexto desnecessário e acelera o agente.

Conceitos-chave:

Sintaxe glob; .env; node_modules; build artifacts.

O que é:

Definir quais ferramentas o Claude Code pode usar sem pedir permissão.

Por que aprender:

Balancear segurança com produtividade.

Conceitos-chave:

allowedTools; deniedTools; permission prompts; trust levels.

O que é:

Variáveis de ambiente que o Claude Code reconhece: ANTHROPIC_API_KEY, CLAUDE_MODEL, etc.

Por que aprender:

Configuração por ambiente sem tocar em arquivos.

Conceitos-chave:

ANTHROPIC_API_KEY; CLAUDE_MODEL; .env local.

O que é:

Diferentes configs para diferentes projetos/contextos.

Por que aprender:

Um projeto pessoal e um corporativo têm necessidades diferentes.

Conceitos-chave:

Settings local vs global; herança de configs.

O que é:

Como usar --model, /model e providers alternativos (Bedrock, Vertex).

Por que aprender:

Flexibilidade para usar o modelo certo para cada tarefa.

Conceitos-chave:

--model flag; CLAUDE_MODEL; Bedrock; Vertex AI.

💾 Memory e Contexto Persistente

Como a memória funciona, o que é context window, compaction e estratégias para não perder contexto.

O que é:

O Claude Code não tem memória mágica. Tudo vive no context window da sessão.

Por que aprender:

Entender o modelo mental evita frustrações com "ele esqueceu".

Conceitos-chave:

Context window = memória de trabalho; CLAUDE.md = memória persistente.

O que é:

/memory salva notas persistentes que o Claude Code lê em toda sessão futura.

Por que aprender:

É o jeito rápido de ensinar preferências sem editar CLAUDE.md.

Conceitos-chave:

/memory; CLAUDE.md gerado; ~/.claude/CLAUDE.md.

O que é:

O context window tem limite fixo (200k tokens). Tudo compete por espaço.

Por que aprender:

Quando o contexto enche, a qualidade das respostas cai.

Conceitos-chave:

Tokens; limite de contexto; degradação gradual.

O que é:

O Claude Code resume automaticamente a conversa quando o contexto enche.

Por que aprender:

Compaction perde detalhes. Saber quando vai acontecer ajuda a se preparar.

Conceitos-chave:

/compact manual; auto-compaction; o que se perde.

O que é:

Estratégias para transferir contexto de uma sessão para outra.

Por que aprender:

Sessões longas degradam. Saber dividir mantém a qualidade alta.

Conceitos-chave:

/resume; summary manual; CLAUDE.md como handoff.

O que é:

Técnicas para salvar o estado do trabalho antes que o contexto encha.

Por que aprender:

Checkpoints previnem perda de progresso em tarefas longas.

Conceitos-chave:

Git commits como checkpoint; notas em CLAUDE.md; TODO files.

Por que aprender:

Scripts wrapper automatizam tarefas repetitivas com IA.

Conceitos-chave:

Wrapper scripts; input via pipe; output parsing.

O que é:

Usar Claude Code dentro do terminal do VS Code e com a extensão oficial.

Por que aprender:

VS Code é o editor mais usado. Integrar bem maximiza produtividade.

Por que aprender:

Sessões sobrevivem a desconexão e permitem múltiplos agentes.

tmux new-session; panes lado a lado; detach/attach.

O que é:

Manter configs do Claude Code em dotfiles versionados.

Por que aprender:

Setup reproduzível em qualquer máquina nova.

Conceitos-chave:

Dotfiles repo; symlinks; bootstrap script.

O que é:

O setup completo que combina tudo: aliases + tmux + VS Code + dotfiles.

Por que aprender:

Conceitos-chave:

Workflow integrado; ritmo diário; evolução incremental.
