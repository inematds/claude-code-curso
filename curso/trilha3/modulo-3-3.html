1

💬 Slack MCP

O Slack MCP conecta o agente aos canais e DMs do seu workspace. Tools tipicas: read_channel (ler mensagens recentes), send_message (enviar mensagem), search_messages (buscar no historico). Precisa de um OAuth Bot Token configurado como variavel de ambiente.

Caso de uso real: voce pede "resuma as ultimas 50 mensagens do canal #engineering" e o agente le, processa e te da um resumo. Ou "manda no #standup: deploy do modulo X concluido" e ele posta por voce.

settings.json (slack)
"slack": {
  "command": "npx",
  "args": ["-y", "@anthropic/mcp-server-slack"],
  "env": {
    "SLACK_BOT_TOKEN": "xoxb-seu-token-aqui"
  }
}

Em 1 frase: Slack MCP = ler, enviar e buscar mensagens do Slack via agente.

2

📆 Google Calendar MCP

Com o Calendar MCP, o agente pode listar eventos (list_events), criar reunioes (create_event), checar disponibilidade (check_availability). Precisa de uma Service Account ou OAuth2 do Google configurado.

Exemplo pratico: "qual minha agenda de amanha?" e o agente puxa os eventos, resume e destaca conflitos. Ou "agenda uma call com Joao amanha as 14h" e ele cria o evento direto.

Em 1 frase: Calendar MCP = ver, criar e gerenciar eventos do Google Calendar via agente.

3

📧 Gmail MCP

O Gmail MCP expoe tools para search_emails, read_email, send_email, create_draft. O agente pode fazer triagem de inbox: ler emails nao lidos, resumir, e ate rascunhar respostas para voce revisar.

Cuidado com permissoes: send_email envia de verdade. Se voce so quer triagem, use create_draft em vez de send, e revise antes de enviar.

Cuidado com send_email

O agente pode enviar emails de verdade. Se nao quiser isso, configure o MCP em modo read-only ou remova a tool send_email das permissoes no settings.json.

Em 1 frase: Gmail MCP = triagem, busca e rascunho de emails direto no agente.

4

📝 Notion/Linear MCP

MCPs para gerenciamento de projeto: o Notion MCP permite criar pages, buscar docs, atualizar databases. O Linear MCP permite criar issues, listar sprints, atualizar status. Ambos precisam de API keys no env.

Cenario: o agente termina uma tarefa de codigo, automaticamente cria uma issue de review no Linear e documenta a mudanca no Notion. Zero trabalho manual de gerenciamento.

Em 1 frase: Notion/Linear MCPs = documentacao e issues automaticas, sem sair do agente.

5

🔗 Combinando multiplos MCPs

O verdadeiro poder do MCP aparece quando voce roda 3-5 servers ao mesmo tempo. O agente ve todas as tools de todos os servers e pode orquestrar entre eles. Exemplo: "le o email do cliente, cria uma issue no Linear e avisa no Slack" = tres MCPs numa unica instrucao.

Cuidado com o tool budget: cada tool que o agente ve consome tokens de contexto (o schema fica no system prompt). Com 50+ tools, o custo de contexto sobe. Mantenha so os MCPs que voce realmente usa no dia a dia.

Em 1 frase: multiplos MCPs simultaneos = o agente orquestra cross-tool sem codigo.

6

⚡ Automacoes do dia a dia

Receitas prontas para copiar e adaptar:

Morning briefing

"Le meus emails nao lidos, mostra minha agenda de hoje, e resume as ultimas mensagens do #general no Slack."

Post-deploy notify

"Manda no Slack #deploys: 'Deploy v2.3 concluido' e cria uma page no Notion com o changelog."

Email triage

"Le os 20 emails mais recentes, classifica por urgencia, e rascunha respostas para os urgentes."

Sprint review

"Lista as issues concluidas no Linear esta semana e gera um resumo para o standup."

Em 1 frase: combine MCPs de produtividade em workflows que rodam com uma unica instrucao.
