🔌 O que sao MCP Servers

O protocolo MCP explicado do zero: arquitetura, schemas, e por que muda tudo.

O que e:

MCP (Model Context Protocol) e o protocolo aberto que conecta LLMs a ferramentas externas via JSON-RPC.

Por que aprender:

E o padrao que o Claude Code usa para todas as integracoes externas.

Conceitos-chave:

JSON-RPC, stdio/SSE transport, handshake de capabilities.

O que e:

O Claude Code e o client; cada MCP e um server que expoe tools via schema.

Por que aprender:

Entender a separacao ajuda a debugar e escolher MCPs.

Conceitos-chave:

Client, server, transport layer, lifecycle.

O que e:

Cada server declara suas tools com inputSchema JSON Schema; o client lista e invoca.

Por que aprender:

E o mecanismo que permite ao agente usar ferramentas sem hardcode.

Conceitos-chave:

tools/list, tools/call, inputSchema, JSON Schema.

O que e:

MCP transforma o agente de "leitor de arquivos" em "operador de sistemas".

Por que aprender:

Sem MCP, o agente so le/escreve files; com MCP, opera DBs, APIs, services.

Conceitos-chave:

Extensibilidade, composicao, padronizacao.

O que e:

Ja existem 200+ MCP servers open source cobrindo DBs, SaaS, DevOps e mais.

Por que aprender:

Saber o que existe evita reinventar a roda.

Conceitos-chave:

Awesome MCP, Smithery, categorias de servers.

O que e:

MCP e como uma API REST, mas otimizada para agentes: schema declarativo, discovery automatico.

Por que aprender:

A analogia facilita entender o modelo mental se voce ja usou APIs.

Conceitos-chave:

REST endpoints vs MCP tools; OpenAPI vs inputSchema.

🚀 Seus Primeiros MCPs

Mao na massa: configurar Filesystem, Git e SQLite no Claude Code.

O que e:

O arquivo .claude/settings.json onde voce declara cada MCP server.

Por que aprender:

Sem isso nenhum MCP funciona; e o primeiro passo pratico.

Conceitos-chave:

mcpServers, command, args, env.

O que e:

Server que expoe operacoes de filesystem como tools MCP.

Por que aprender:

E o MCP mais basico e ilustra perfeitamente o pattern.

Conceitos-chave:

read_file, write_file, list_directory, sandboxing.

O que e:

Server MCP para operacoes Git: status, diff, commit, branch.

Por que aprender:

Permite ao agente versionar codigo sem voce intervir manualmente.

Conceitos-chave:

git_status, git_commit, git_diff, permissoes.

O que e:

Server que da ao agente acesso SQL a bancos SQLite.

Por que aprender:

Demonstra como MCPs podem dar acesso a dados estruturados.

Conceitos-chave:

query, execute, schema inspection, read-only mode.

O que e:

Como verificar se o MCP esta conectado e as tools aparecem no agente.

Por que aprender:

Sem teste voce nao sabe se a config esta certa.

Conceitos-chave:

/mcp, tool listing, invocacao manual, logs.

O que e:

Diagnosticar erros comuns: server nao inicia, timeout, permission denied.

Por que aprender:

90% dos problemas com MCP sao de config; saber debugar economiza horas.

Conceitos-chave:

stderr logs, npx vs global install, PATH, env vars.

🛠️ Criando seu Proprio MCP Server

Do zero ao deploy: anatomia, SDK TypeScript, tools, handlers e distribuicao.

O que e:

Um server MCP e um processo que fala JSON-RPC via stdio e expoe tools.

Por que aprender:

Entender a anatomia e pre-requisito para criar o seu.

Conceitos-chave:

Processo, stdin/stdout, initialize, tools/list handler.

O que e:

O pacote @modelcontextprotocol/sdk que abstrai o protocolo pra voce.

Por que aprender:

Com o SDK, voce cria um server funcional em 30 linhas.

Conceitos-chave:

McpServer class, StdioServerTransport, tool(), resource().

O que e:

Cada tool tem nome, descricao e um inputSchema (JSON Schema) que o agente usa.

Por que aprender:

Um schema bem feito = o agente acerta os parametros de primeira.

Conceitos-chave:

name, description, inputSchema, required fields, types.

O que e:

A funcao que recebe os args do agente, executa a acao e retorna o resultado.

Por que aprender:

E onde a logica real vive; o handler e o "corpo" da tool.

Conceitos-chave:

CallToolResult, content array, text/image responses, error handling.

O que e:

Testar o MCP server localmente com o MCP Inspector e com Claude Code.

Por que aprender:

Um server sem testes quebra em silencio; o inspector facilita muito.

Conceitos-chave:

MCP Inspector, manual JSON-RPC, integration tests.

O que e:

Como empacotar e publicar seu MCP server (npm, Docker, Smithery).

Por que aprender:

Compartilhar com o time ou a comunidade multiplica o impacto.

Conceitos-chave:

npm publish, npx entry point, Docker image, Smithery listing.

🌐 Ecossistema e Marketplace

Onde encontrar, avaliar e compor MCP servers de qualidade.

O que e:

O repositorio awesome-mcp-servers no GitHub: a lista curada da comunidade.

Por que aprender:

Ponto de partida para descobrir MCPs prontos de alta qualidade.

Conceitos-chave:

Categorias, stars, manutencao ativa, oficiais vs community.

O que e:

Smithery.ai: o npm/marketplace para MCP servers com install one-click.

Por que aprender:

Instalar MCPs via Smithery e mais rapido que configurar manualmente.

Conceitos-chave:

smithery install, verified servers, usage stats.

O que e:

Criterios para avaliar se um MCP server e confiavel antes de usar.

Por que aprender:

MCP ruim pode travar, vazar dados ou dar respostas erradas.

Conceitos-chave:

Commits recentes, issues abertas, documentacao, testes, licenca.

O que e:

Patterns para combinar 3-8 MCPs num setup coeso e produtivo.

Por que aprender:

O poder do MCP aparece na composicao, nao em servers isolados.

Conceitos-chave:

Tool budget, naming conflicts, priority routing.

O que e:

Riscos de MCPs maliciosos e como se proteger (sandbox, audit, least-privilege).

Por que aprender:

Um MCP roda codigo na sua maquina; confiar cegamente e perigoso.

Conceitos-chave:

Code review, pinned versions, sandboxing, permission scopes.

O que e:

Tendencias: remote servers, auth federada, streaming, multi-modal tools.

Por que aprender:

Saber a direcao ajuda a investir no que vai durar.

Conceitos-chave:

Remote MCP, OAuth2, Streamable HTTP, tool composability.
