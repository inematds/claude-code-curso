4.1~40 min

Subagentes — Conceitos

O que sao subagentes, a Task tool e quando usar paralelismo.

O que e:

Subagentes sao instancias do Claude Code lancadas pelo agente principal para executar tarefas isoladas.

Por que aprender:

Permite escalar trabalho sem poluir o contexto do agente principal.

Conceitos-chave:

Pai-filho; isolamento de contexto; delegacao.

O que e:

A ferramenta interna do Claude Code que cria subagentes com prompt e contexto proprio.

Por que aprender:

E a interface primaria para orquestrar trabalho paralelo.

Conceitos-chave:

Task(prompt); retorno de resultado; escopo limitado.

O que e:

Subagentes podem ter niveis diferentes de permissao: leitura, escrita ou background.

Por que aprender:

Escolher o tipo certo evita conflitos e reduz risco.

Conceitos-chave:

Principio do menor privilegio; agentes especializados.

O que e:

O pai envia instrucao e contexto; o filho retorna resultado textual ao terminar.

Por que aprender:

Entender o fluxo evita perda de informacao entre agentes.

Conceitos-chave:

Prompt de entrada; retorno estruturado; sem estado compartilhado.

O que e:

Worktrees do git permitem que cada subagente trabalhe em arquivos sem conflitar com outros.

Por que aprender:

Sem isolamento, agentes paralelos pisam uns nos outros.

Conceitos-chave:

git worktree; branch temporaria; merge posterior.

O que e:

Subagentes tem overhead de contexto e custo. Nem toda tarefa justifica paralelismo.

Por que aprender:

Evita gastar tokens desnecessariamente em tarefas simples.

Conceitos-chave:

Overhead vs ganho; tarefas independentes; complexidade minima.

Paralelismo Real

Lance multiplos agents, colete resultados e lide com erros em paralelo.

O que e:

Disparar varias Task tools simultaneamente para tarefas independentes.

O que e:

Identificar tarefas que nao dependem umas das outras para execucao simultanea.

O que e:

Usar git worktree add para criar espacos de trabalho isolados por agente.

O que e:

Executar tarefas longas em background enquanto o agente principal continua.

O que e:

Reunir os outputs de todos os subagentes e consolidar num resultado final.

O que e:

Estrategias para lidar com falha parcial: retry, skip ou abort.

Patterns de Multi-Agent

Padroes arquiteturais: fan-out/in, pipelines, specialists e cost control.

O que e:

Distribuir trabalho para N agentes e reunir resultados num unico ponto.

O que e:

Encadear agentes onde o output de um alimenta o proximo.

O que e:

Um agente explora o codebase, outro planeja, outro executa. Separacao de preocupacoes.

O que e:

Agentes com prompts especificos para testes, docs, lint ou seguranca.

O que e:

Patterns de retry, fallback e circuit-breaker adaptados para agentes de IA.

O que e:

Limitar tokens por subagente, usar modelos menores para tarefas simples, medir ROI.

Git com IA — Versionamento

Commits semanticos, staging inteligente e merge conflicts com Claude.

O que e:

Claude analisa o diff e gera mensagens de commit no padrao conventional commits.

O que e:

Claude seleciona arquivos logicamente relacionados para commits atomicos.

O que e:

Pedir ao Claude para revisar o diff antes de confirmar, pegando bugs e inconsistencias.

O que e:

Claude entende o contexto de ambos os lados e resolve conflitos semanticamente.

O que e:

Usar branches curtas e focadas que facilitam o trabalho do agente.

O que e:

Proteger segredos (.env, keys) e evitar que o agente commite arquivos sensiveis.
